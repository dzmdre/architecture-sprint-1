Разбитие монолитного приложения фронтенд на микрофронтенды с использованием Single SPA
==========================================================================================================================================================

Введение
----------------

I этап: Оценка текущего состояния
------------------------------------------------------------

1. Оценить сложность и масштаб монолитного фронтенд-приложения.
   Приложение вполне может существовать в монолитном виде, но с ростом команды и функционала, поддержка и развитие монолита становится сложнее.
   По условиям задания разделим монолит на микрофронтенды.
2. Сформулировать цели и преимущества перехода на микрофронтенды.
   Основной плюс независимая разработка и деплой отдельных частей приложения как и у микросервисов на бэкенде.
   Основной минус - усложнение архитектуры и коммуникации между частями приложения(особенно важно учитывать искусственное усложнение проекта, как здесь).
3. Оценить опыт команды в работе с микрофронтендами и Single SPA.

---

II этап: Определить бизнес-функции, которыми будут заниматься разные команды

1. Принципы Domain-Driven Design (DDD) для выделения бизнес-функций.
   Функциональность Mesto

- Загрузка фотографий
- Удаление фотографий
- Сбор и учёт лайков под фото
- Создание профиля и его редактирование

   - Микрофронтенд профиля пользователя (просмотр и редактирование профиля)

III этап:  Выбрать метод реализации микрофронтендов

- Build time: Серверная компоновка
- Run time: Клиентская компоновка, Серверная компоновка, Гибридный подход
  IV этап: Выбрать инструмент для создания микрофронтендов
- Single SPA и Webpack Module Federation — самые популярные фреймворки для микрофронтендов. Но есть и другие варианты.
  Вот топ альтернативных фреймворков для создания микрофронтендов. Чем выше фреймворк в списке, тем он популярнее:
- Piral — позволяет создавать модульные фронтенд-приложения, которые расширяются во время выполнения с помощью разрозненных модулей.
- Luigi — помогает создавать модульные, расширяемые, масштабируемые и целостные пользовательские интерфейсы и веб-приложения.
- OpenComponents — упрощает создание и развёртывание компонентов фронтенда.
- Voltran — фреймворк для микрофронтендов, разработанный командой Hepsiburada Technology Team.
- Qiankun — позволяет создавать веб-приложения на базе архитектуры микрофронтендов. Вдохновлён Single SPA и основан на этом фреймворке.
- Bit — комплексное решение для создания составных приложений (composable applications). Упрощает создание, сопровождение и повторное использование независимых компонентов.
- PuzzleJs — микрофреймворк для создания масштабируемых и очень быстрых веб-сайтов.
